Dijkstra’s Algorithm for Disaster Relief

Home Algorithm Simulation Priority Workflow Results Conclusion

Objective

To find the shortest and fastest delivery routes between disaster-affected zones using Dijkstra’s Algorithm.

Algorithm Overview

Dijkstra’s Algorithm is a graph-based shortest path algorithm that identifies the minimum distance from the central relief center to all affected zones, ensuring faster delivery of relief supplies.

Algorithm Steps

  1. Mark all zones as unvisited and assign infinite distance.
  2. Set distance of the source zone (relief center) to 0.
  3. Choose the unvisited zone with the smallest distance.
  4. Update distances of its neighboring zones.
  5. Repeat until all zones are visited.

Sample C Implementation

#include <stdio.h>
#include <limits.h>
#define V 5

int minDistance(int dist[], int sptSet[]) {
  int min = INT_MAX, min_index = -1;
  for (int v = 0; v < V; v++) {
    if (!sptSet[v] && dist[v] <= min) {
      min = dist[v];
      min_index = v;
    }
  }
  return min_index;
}

void dijkstra(int graph[V][V], int src) {
  int dist[V], sptSet[V];
  for (int i = 0; i < V; i++) {
    dist[i] = INT_MAX;
    sptSet[i] = 0;
  }
  dist[src] = 0;

  for (int count = 0; count < V - 1; count++) {
    int u = minDistance(dist, sptSet);
    sptSet[u] = 1;
    for (int v = 0; v < V; v++) {
      if (!sptSet[v] && graph[u][v] && dist[u] != INT_MAX &&
          dist[u] + graph[u][v] < dist[v]) {
        dist[v] = dist[u] + graph[u][v];
      }
    }
  }

  printf("Zone\tShortest Distance from Relief Center\n");
  for (int i = 0; i < V; i++)
    printf("%d\t\t%d\n", i, dist[i]);
}

int main() {
  int graph[V][V] = {
    {0, 10, 0, 30, 100},
    {10, 0, 50, 0, 0},
    {0, 50, 0, 20, 10},
    {30, 0, 20, 0, 60},
    {100, 0, 10, 60, 0}
  };
  dijkstra(graph, 0);
  return 0;
}

Complexity Analysis

  • Time Complexity: O(V²) for adjacency matrix; O((V + E) log V) using min-heap.
  • Space Complexity: O(V²) for graph, O(V) for distance and visited arrays.

Limitations

  • Cannot handle negative edge weights.
  • Assumes static conditions — does not adapt to real-time changes.
  • Ignores urgency factor unless prioritized separately.
